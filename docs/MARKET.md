# Pomo 市場性判定（2026-06-11）

調査ワークフロー（収益実態・App Store 要件・Raycast 品質 DNA・DB 選定・管理画面設計の5軸 → 強気/弱気の独立両論 → 中立判定、エージェント8体）の結論。

## 判定: 条件付きで売れる（確信度: 中）

**結論の骨子:**
- **¥300・日本語圏・Mac App Store 先行は「死路」と実証済み** — monora が4ヶ月でレビュー5件未満。この戦略を取るなら「売れない」判定になる
- **$19 買い切り・英語圏・直販は実証された生存路** — LookAway（$19、開発者層向け眼精疲労アプリ）が約4,000本販売。Sensei は $300K ARR で「Mac ユーティリティは支払い意欲が高い」と明言
- ただし「ポモドーロ × Claude Code 連携」は LookAway より狭いニッチ。**収益の中央値は年¥15万〜80万（確率加重期待値 約¥50万）**。時給換算では割に合わない。Swift/出荷経験・ポートフォリオ価値込みで投資判断すべき

## 売る前の絶対条件（P0）

1. ✅ **API トークン認証**（2026-06-11 実装済み。`~/Library/Application Support/Pomo/token`、Bearer 認証）
2. **需要の事前検証**: Claude Code 連携のデモ動画を X/HN に出す。反応が閾値未達なら販売を中止し「無料/OSS + ポートフォリオ」路線へ転換する**撤退基準を先に決める**
3. Xcode プロジェクト化 + Developer ID 署名 + notarytool 公証
4. Paddle 開設 + ライセンスキー + トライアルロジック
5. Sparkle 自動更新
6. UI 英語化 + 英語 LP + デモ動画
7. **会議・画面共有中のオーバーレイ自動延期**（即アンインストール級クレームの予防）— P1 だが出荷前必須

## 技術判定

- **DB は導入しない（JSONL 維持で確定）**: 年1.4MB・追記専用・集計は数ms。BigQuery のネイティブロード形式が newline-delimited JSON なので、現ファイルがそのままエクスポート成果物。SQLite/SwiftData/Core Data は全部この規模では過剰。移行トリガー: 50万行超 or パース300ms超 or 複数プロセス書込 or 全文検索が必要になったら GRDB 一択
- **MAS 化の地雷**: サンドボックスで sessions.jsonl が `~/Library/Containers/` 配下に移動し、外部からの直読みが壊れる → 連携は HTTP API 経由に一本化してから

## この判定が破綻しうる条件

- 需要検証（デモ動画の反応）が閾値未達 → 即「無料/OSS」路線へ。この場合も Swift/出荷経験という投資回収は成立する

- 英語圏ローンチに必要な英語での発信・サポートのコストを負えない場合、中央値はさらに下がる
