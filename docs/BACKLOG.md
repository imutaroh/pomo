### DB 調査結果（2026-06-12、Web 調査済み・根拠 URL はエージェントログ参照）

- **ライブラリ本命は GRDB.swift**（v7.11.0 / 2026-06 リリース、8.4k★、純 SPM 対応、FTS5 デフォルト有効）。SwiftData は SPM executable で公式想定外＋FTS なし、SQLite.swift はメンテ低調
- **構成: JSONL は唯一の書込先のまま**（BigQuery ロード形式・append-only を維持 = 既定の「DB やらない」決定と整合）。SQLite は「壊れたら JSONL から全再構築できる使い捨てインデックス」。起動時＋セッション追記時に増分更新
- メモ全文検索は FTS5 `tokenize='trigram'`。**2文字以下のクエリは0件になる制限**があるため `LIKE '%…%'` フォールバック必須（数万行なら十分速い）
- 書き手はアプリ単一プロセスのみ（localhost API は 2026-06-13 に廃止）なので SQLite 単一書き手問題は発生しない
- **破綻条件**: ①メモの後編集・タグ付けを SQLite だけに保存したくなった瞬間、「JSONL が正」の前提が崩れる → SQLite を一次ストアに昇格し BigQuery 向けはエクスポートに反転 ②二字熟語検索の精度要求が高まると trigram では不足（lindera 導入 = 配布複雑化）

## やらない（調査で根拠が確定）

| 項目 | 理由 |
|------|------|
| アカウント・クラウド同期・サブスク課金 | HN で最頻出の称賛が「アカウント不要」。Session は「機能は素晴らしいが価格が silly」と切り捨てられた。ローカルオンリー無料は欠点ではなく堀 |
| スキップ不能の強制ロック休憩（デフォルト） | Opal の事例: ユーザーは遵守ではなく回避策探しに走る。PNAS 研究でも最強要素は「自分で閉じる選択肢」 |
| ストリーク・ヒートマップ・ゲーミフィケーション | Forest の罪悪感メカニクスは「飽きる・長期効果弱い」が定着。ヒートマップの空白セルはストリークの変装。no-guilt 方針と正面衝突 |
| AI コーチ・割り込み検知・生産性スコア・ムード入力 | Centered（アプリ切替検知＋音声ナッジ）は「推測しすぎ・侮辱的」と酷評され倒産。判定・評価はストリークフリー思想と相反 |
| 単体 stdio MCP サーバー / URL スキーム / localhost API 全般 | localhost API は App Sandbox（MAS 必須）と衝突するため 2026-06-13 に廃止。復活させない。MCP もコンテキストを大量消費し、エージェント連携より Sandbox 適合・配布を優先する判断 |
